# A股季报成长验证系统

## 1. 这个工具解决什么问题

本工具用于筛选和跟踪一类股票：

- 所属行业不是 AI/TMT 热门方向；
- 因行业 beta 或市场风格原因被无差别下杀；
- 当前处于低位、低估区域；
- 财务上仍体现较好的成长性；
- 定期报告文本中能进一步验证成长是否延续、预期是否更明朗。

工具输出的是研究候选池，不是买卖建议，不自动交易，也不生成仓位指令。

## 2. 当前已经实现的能力

### 第一阶段：量化初筛

使用 Tushare 拉取沪深 A 股基础信息、估值、行情、财务指标，生成候选池。

默认过滤：

- 排除 ST、退市股；
- 排除北交所；
- 排除上市未满 2 年；
- 排除 AI/TMT 高暴露行业或关键词；
- 排除 `-U` 未盈利公司；
- 要求扣非利润为正；
- 要求有低位错杀特征。

核心分数：

- `growth_score`：财务成长分；
- `valuation_score`：估值分；
- `mispricing_score`：错杀/低位分；
- `quality_score`：质量与风险分；
- `total_score`：第一阶段综合分。

### 第二阶段：定期报告文本分析

对初筛通过的股票下载正式定期报告 PDF，解析文本，截取经营、展望、风险相关内容，再交给 LLM 分析。

当前支持：

- 一季报：`20260331`
- 半年报：`20260630`
- 后续也兼容三季报、年报标题识别逻辑

LLM 判断：

- 成长是否延续；
- 预期是否更明朗；
- 成长质量是否健康；
- 风险是否恶化；
- 是否继续跟踪、降权观察或剔除。

```text
.
├─ .venv/                         本项目固定 Python 环境

│  ├─ reports/

├─ refresh_reports.ps1            下载和解析定期报告

└─ run_dashboard.ps1              启动网页 Dashboard
```

## 4. 配置文件

密钥配置在：

```text
.env.txt
```

需要包含：

```env
TUSHARE_TOKEN=你的tushare_token
LLM_API_KEY=你的llm_key
LLM_BASE_URL=https://api.deepseek.com
LLM_MODEL=你的模型名
```

程序已经兼容 `.env.txt`，不用强制改名为 `.env`。

## 5. 如何运行

由于 Windows 当前禁止直接运行 `.ps1`，推荐使用 `-ExecutionPolicy Bypass`。

### 跑一季报测试

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\screen_candidates.ps1 -Period 20260331 -TargetCount 30
powershell -NoProfile -ExecutionPolicy Bypass -File .\refresh_reports.ps1 -Period 20260331
powershell -NoProfile -ExecutionPolicy Bypass -File .\analyze_reports.ps1 -Period 20260331
```

### 跑半年报跟踪

半年报披露后运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\screen_candidates.ps1 -Period 20260630 -TargetCount 80
powershell -NoProfile -ExecutionPolicy Bypass -File .\refresh_reports.ps1 -Period 20260630
powershell -NoProfile -ExecutionPolicy Bypass -File .\analyze_reports.ps1 -Period 20260630
```

### 启动 Dashboard

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\run_dashboard.ps1
```

也可以直接用固定 Python 环境：

```powershell
.\.venv\Scripts\python.exe -m stock_screener.screen_market --period 20260331 --target-count 30
.\.venv\Scripts\python.exe -m stock_screener.refresh_reports --period 20260331
.\.venv\Scripts\python.exe -m stock_screener.analyze_reports --period 20260331
.\.venv\Scripts\python.exe -m streamlit run app.py
```

```text

```

### 文本分析结果

```text

data\outputs\report_analysis_20260630.xlsx
```

用途：最终研究表，包含财务分、文本分、LLM 结论和原文证据。

## 7. report_analysis 字段说明

| 字段 | 含义 |
|---|---|
| `ts_code` | 股票代码 |
| `name` | 股票名称 |
| `industry` | 行业 |
| `total_score` | 第一阶段量化总分 |
| `growth_score` | 第一阶段财务成长分 |
| `financial_continuation_score` | 财务延续性分 |
| `text_confirmation_score` | 文本确认分 |
| `verification_total_score` | 财务和文本综合验证总分 |
| `candidate_status` | 最终状态：`A_confirmed`、`B_watch`、`C_reject` |
| `growth_continues_score` | LLM 判断成长是否延续 |
| `visibility_score` | LLM 判断预期是否更明朗 |
| `quality_of_growth_score` | LLM 判断成长质量 |
| `risk_deterioration_score` | LLM 判断风险恶化程度，越高风险越大 |
| `text_verdict` | LLM 文本结论 |
| `reasoning` | 简短分析逻辑 |
| `evidence_quotes` | 定期报告原文证据 |
| `llm_error` | LLM 调用错误，空值代表成功 |
| `section_path` | 被送入 LLM 的文本片段路径 |
| `section_chars` | 文本片段字符数 |
| `evidence_json` | 原文证据 JSON |

## 8. 状态怎么理解

```text
A_confirmed
```

财务和文本都比较强，成长延续性与预期明朗度较好，优先跟踪。

```text
B_watch
```

有成长证据，但可能存在现金流、费用、价格、订单能见度等问题，需要继续观察。

```text
C_reject
```

成长证据不足、文本不支持、风险恶化，或预期不明朗，暂时剔除。

## 10. 注意事项

- 本工具是研究辅助，不是投资建议。
- Tushare 接口权限会影响公告和财务数据获取。
- DeepSeek 或其他 LLM 后台用量统计可能有延迟。
- 如果要核对 token 消耗，后续可以把 API 返回的 `usage` 字段写入 Excel。
- 缓存目录建议保留，否则重复运行会重新消耗 Tushare 接口调用。
